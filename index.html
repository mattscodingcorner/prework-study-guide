HTML

  • The head element contains informaiton about the webpage
  • The body element represents the visible content shown to the user.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on the webpage.
  • Semantic elements describes its meaning to both the browser and the developer.

CSS

  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • External CSS style sheets are usually preferred in most cases because it allows devs to keep all their CSS rules in a seperate file like this guide.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and swithces to it
  • git pull origin main: fetch and download content from a remote repo and immediately update the local repo to match the content.
  • A good habit to always pull the base branch into your feature branch before pushing up any code.
  • git add -A: used to add the changed files in our working directory to the staging area of Git.
  • git commit -m "notes about changes": this will commmit your changes to the repo, helpful to add notes about what you changed/worked on.
  • -m flag to associate a message with our commit. Commit messages should be short and descriptive.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An Array is a single variable that is used to hold a group of data. Typically used to hold data that is related in some way.
  • Arrays are zero-indexed and sequential, first item in any array has an index of 0 not 1.
  • A function is simply a set of instrcutions that tells the computer on how to perform a certain task.
